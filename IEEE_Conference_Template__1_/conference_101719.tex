\begin{abstract}
Existing test prioritization tools for self-driving car simulations reduce road geometry to a handful of aggregate statistics, discarding the sequential structure that makes a road dangerous. We present \textbf{RoadFury}, which preserves the full geometric sequence: a 10-channel feature matrix processed by a Pre-LN Transformer Encoder with Stochastic Weight Averaging. A systematic comparison of four paradigms---LLMs, GNNs, CNNs, and Transformers---reveals that only sequential encoding consistently outperforms random ordering. Deployed for the ICST~2026 SDC Testing Competition~\cite{b8}, RoadFury achieves APFD~=~\textbf{0.804\,$\pm$\,0.012}.
\end{abstract}

test prioritization, self-driving cars, Transformer, stochastic weight averaging, road geometry

\section{Introduction}

Testing self-driving cars in high-fidelity simulators like BeamNG.tech is essential but expensive~\cite{b1,b2}: suites easily grow to tens of thousands of tests, each requiring tens of seconds to minutes. \emph{Test prioritization} reorders tests so that failure-revealing cases run first, measured by APFD~\cite{b4}:
\vspace{-3pt}
\begin{equation}
\text{APFD}(\pi) = 1 - \frac{\sum_{i=1}^{m} TF_i}{n \cdot m} + \frac{1}{2n}
\end{equation}
where $n$ is total tests, $m$ failing tests, and $TF_i$ the rank of the $i$-th fault (1.0: all failures first; 0.5: random). Existing tools such as SDC-Scissor~\cite{b1,b2} and ITEP4SDC reduce each road to three aggregate statistics---segment length, angle change, curvature---discarding the \emph{sequential structure} that makes a road dangerous: the precise ordering of straights followed by sudden hairpins.

This raises a natural question: \emph{what is the right representation for road geometry?} We explored four paradigms---zero-shot LLMs~\cite{b16}, GNNs~\cite{b21}, visual CNNs~\cite{b5}, and sequential Transformers~\cite{b11}---and found that only the last consistently surpasses random ordering. This led to \textbf{RoadFury}: a 10-channel geometric feature representation processed by a Pre-LN Transformer~\cite{b11} with SWA~\cite{b6}. Prior work supports these choices: Transformers outperform classical classifiers on sequential inputs~\cite{b15}; early-stage ML can predict AI-artifact outcomes from structural features~\cite{b14}; SWA finds wider optima~\cite{b6}; Yoo and Harman~\cite{b4} survey regression testing.

\section{Approach}

\subsection{Road Normalization and Feature Extraction}

Each test case is a variable-length sequence of 2D road control points. We resample all sequences to $L{=}197$ points via linear interpolation and extract 10 intrinsic geometric channels (Table~\ref{tab:features}), yielding a $(197{\times}10)$ translation- and rotation-invariant feature matrix, z-normalized with training-set statistics.

\subsection{RoadTransformer Architecture}

The architecture (Fig.~\ref{fig:arch}) reads a road as a sequence, much like a Transformer reads a sentence. Each 10-dimensional point vector is projected to 128 dimensions via a linear layer with layer normalization and GELU; a learnable \texttt{[CLS]} token is prepended and sinusoidal positional embeddings are added, yielding 198 tokens. These pass through a four-layer Pre-LN Transformer Encoder~\cite{b11} (8 heads, FFN dim 512, GELU, dropout 0.1). The Pre-LN variant normalizes \emph{before} each sub-layer, stabilizing training and avoiding gradient issues. The \texttt{[CLS]} output then passes through an MLP head (LN, 64-unit hidden layer with GELU and 20\% dropout, sigmoid) to produce a failure probability $\hat{y}{\in}[0,1]$. The full model has {\raise.17ex\hbox{$\scriptstyle\sim$}}400K parameters.

\subsection{Training and Deployment}

Training uses Adam (lr\,=\,$10^{-3}$), binary cross-entropy, 75 epochs on 28,804 samples. The key ingredient is \emph{SWA}~\cite{b6}: from epoch~50, model weights are averaged over the remaining 25 epochs, finding a flatter loss basin that generalizes better to unseen road distributions. At inference, all $n$ tests are scored in one batched forward pass and sorted by descending $\hat{y}$. The tool is containerized as a gRPC service (\texttt{Name}, \texttt{Initialize}, \texttt{Prioritize}).

\section{Evaluation}

We evaluate on SensoDat~\cite{b7} (36,006 labeled BeamNG.tech tests, 80/20 split; 956-test competition set held out). All results average 30 stratified random trials.

\textbf{Why not just any ML model?} Table~\ref{tab:results} compares four paradigms, each encoding road geometry differently.

\caption{Multi-Paradigm Comparison (956 comp.\ tests, 30 trials)}
\vspace{-2pt}

ExtendedCNN-10ch         & 0.768\,$\pm$\,0.014 & 10-ch CNN, 28K data \\

The results reveal a clear hierarchy driven by \emph{how much sequential structure each paradigm preserves}. LLMs score \emph{below} random: they can discuss curvature in the abstract but cannot judge whether a specific value is dangerous. GNNs~\cite{b21} capture local neighborhoods via KNN edges but discard point ordering---yet a sharp turn after a long straight is far more dangerous than the same turn after a gradual curve. Visual CNNs~\cite{b5} treat roads as images, losing numeric precision; even fused with 44 handcrafted features they plateau at 0.673. The most revealing comparison is ExtendedCNN-10ch vs.\ pure CNN baselines: our 10-channel features in a simple 1D-CNN already reach 0.768, showing that \emph{what} you measure matters more than \emph{how} you model it. RoadFury's Transformer attends to all 197 points simultaneously, learning which curvature \emph{transitions} predict failures, yielding the best APFD of 0.804.

\textbf{Ablation.} Seven single-factor modifications over the base Pre-LN Transformer (0.790): multi-scale conv stems, stochastic depth~\cite{b20}, alternative pooling, mixup~\cite{b19}, and TTA all \emph{degraded} performance; focal loss~\cite{b18} gave a marginal gain; only SWA~\cite{b6} consistently helped (+0.014). The architecture is already well-matched to the task---regularization in weight space, not architectural complexity, is the key.

%=====================================================================
\section{Conclusion}

\emph{How} road geometry is represented matters more than model scale or training tricks. Language models, graph networks, and CNNs each discard the sequential curvature dynamics cars actually experience; a Pre-LN Transformer preserves them, achieving APFD\,=\,0.804. Future work targets online oracle adaptation and multi-scale SWA scheduling. Code: \url{https://github.com/chisngyen/sdc-test-prioritization-novel}.
